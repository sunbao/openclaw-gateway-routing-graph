# Contributing

感谢你关注 `openclaw-gateway-routing-graph`。

这个仓库是一个独立的开源项目，目标是以外部监控工具的方式，为 OpenClaw 网关提供中文优先、业务优先的实时链路可视化能力。

## 贡献前先了解

提交改动前，请先阅读以下文件：

- `README.md`
- `CHANGELOG.md`
- `RELEASE_NOTES.md`

## 项目原则

请优先遵守以下约束：

- 保持独立项目形态，不修改 OpenClaw 主工程源码
- 始终按最新行为维护，不保留历史兼容层
- 不做兜底式规则兼容或旧语法兼容
- 优先提升业务可读性，再考虑技术细节
- 技术细节默认下沉，不要破坏现有业务板块的可见性

## 适合贡献的方向

欢迎以下类型的贡献：

- 图面可读性优化
- 中文文案和业务信息表达优化
- 新的业务字段展示
- 会话、渠道、工具链路的展示增强
- 连接异常和错误提示优化
- 文档、示意图、架构图、发布说明完善

## 暂不接受的方向

以下改动请不要直接提交：

- 把监控逻辑直接耦合进 OpenClaw 主工程
- 为旧规则语法增加兼容层
- 默认展示完整敏感消息正文或完整工具输出
- 未经讨论的大规模重构或 UI 推倒重做

## 本地开发

```bash
npm install
npm run dev
```

构建验证：

```bash
npm run build
```

## 提 Issue 建议

提 Issue 时，建议带上这些信息：

- 使用的是观察器模式还是网关直连模式
- 页面里填写的地址类型（HTTP/SSE 或 WebSocket）
- 预期看到什么
- 实际看到什么
- 是否有浏览器报错、连接提示或截图

如果问题和图面相关，附一张截图会很有帮助。

## 提 Pull Request 建议

提交 PR 时请尽量做到：

- 一个 PR 聚焦一个主题
- 说明修改目的和范围
- 说明是否影响图面、标签规则或连接方式
- 如果改了界面，附截图
- 如果改了运行方式或用户可见行为，同步更新文档

## 文档要求

如果你的改动影响以下任一项，请同步更新 README 或相关说明：

- 运行方式
- 连接方式
- 发布方式
- 标签规则
- UI 结构或主要板块含义

## 发布说明

当前项目已经有：

- `LICENSE`
- `CHANGELOG.md`
- `RELEASE_NOTES.md`

如果改动已经达到可对外说明的程度，请同步补充 release 相关文案。

## 交流方式

如果你的改动会影响：

- 主视图布局
- 标签规则结构
- 图面交互模式
- 连接模型

建议先开 Issue 或 PR 说明方案，再进入实现。
