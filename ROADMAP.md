# Roadmap

> 当前仓库已经完成 V1 可运行版本。  
> 下一阶段重点不再是“把图画出来”，而是从“看见链路”走向“发现风险、理解风险、控制风险”。

## 当前版本：V1 已完成

当前已经具备：

- 独立仓库、独立部署、独立运行
- 观察器模式和网关直连模式
- 中文优先的业务驾驶舱
- 实时链路图、入口消息、出口消息、活跃会话
- 技术事件明细、健康检查、同源 WebSocket 代理
- 标签规则可视化配置

## 下一阶段总方向

后续将围绕三条主线推进：

1. **图面继续优化**：更稳、更清晰、更像业务驾驶舱  
2. **风险治理能力**：从监控走向告警、审核、策略控制  
3. **规则中心能力**：从静态标签走向可运营的规则体系

## Phase 2：风险治理规划

> 启动条件：**仓库 Star > 100**  
> 在达到该条件前，先完成需求沉淀、方案设计和 OpenSpec，不急于进入开发。

### 2.1 出口风险告警

目标：

- 发现即将发往渠道的高风险回复
- 先告警，不直接阻断

规划内容：

- 敏感词 / 风险词命中
- 隐私泄露检测
- 渠道出口风险高亮
- 风险事件列表和命中原因展示

### 2.2 出口软拦截

目标：

- 对高风险出口消息做“拦截 / 审核 / 放行”

规划内容：

- 命中规则后进入待审核
- 支持仅告警 / 拦截 / 遮罩后放行
- 记录操作人、时间、原因

### 2.3 入口风险控制

目标：

- 对进入 OpenClaw 的消息做前置风险过滤

规划内容：

- 黑名单用户 / 群 / 渠道拦截
- 入口关键词拦截
- 越权 / 越狱 / 攻击型指令识别
- 会话级、渠道级、账号级控制

### 2.4 规则中心

目标：

- 把规则从“零散配置”提升为“可运营系统”

规划内容：

- 规则分类：关键词、正则、黑名单、隐私规则
- 规则优先级
- 白名单例外
- 命中记录与审计视图
- 启停开关

## Phase 3：策略执行层

目标：

- 在不修改 OpenClaw 主工程的前提下，增加可控的策略执行能力

规划方向：

- Sidecar / Policy Proxy 架构
- 入口前置控制
- 出口回写前置控制
- 审核队列
- 风险策略命中后的统一动作模型

## Phase 4：更强的运营能力

规划方向：

- 规则模板市场
- 组织级策略包
- 多环境规则同步
- 告警通知集成
- 审计导出

## 当前优先级判断

如果真的进入开发，优先级建议如下：

1. **出口隐私泄露告警**
2. **出口高风险内容软拦截**
3. **入口黑名单与关键词拦截**
4. **规则中心**
5. **人工审核流**

## 不变原则

即使进入下一阶段，以下原则仍然保持不变：

- 不把监控和控制逻辑硬塞进 OpenClaw 主工程
- 不为历史 UI 和历史规则写兼容层
- 先做业务可理解的能力，再做复杂技术细节
- 每个阶段都必须能单独演示、单独验收、单独上线
